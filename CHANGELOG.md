### Added

- `notebooks/6.1_intent_classification.ipynb`: fine-tunes `indobenchmark/indobert-base-p1` (`AutoModelForSequenceClassification`, `p1` over `p2` for faster inference since intent runs on every utterance) on `text_normalized` from `data/normalized/intent_dataset_normalized.jsonl` (2060 records, 10-way intent labels already present, no labeling stage needed). `train_test_split` (scikit-learn, stratified by `intent`, `random_state=42`) splits 80/10/10 into `data/intent/{train,val,test}.jsonl` + `label_map.json`. Plain unweighted `Trainer` (`learning_rate=2e-5`, `warmup_ratio=0.1` linear, 5 epochs, batch size 16, `load_best_model_at_end` on macro-F1) — imbalance here is mild (1.3x max: disfluency-augmented `order_create` examples from `3.1` land at 260 vs 200/class elsewhere), unlike the weighted-loss disfluency/NER stages. `compute_metrics` reports accuracy + macro-F1 via `evaluate`; test-set eval adds per-intent `classification_report` and prints up to 15 misclassified examples. Saves to `models/indobert-intent-final`. `predict_intent(text)` reloads the saved model standalone and returns predicted intent plus per-class softmax probabilities. Not yet executed — no run metrics recorded.
- `notebooks/3.1_disfluency_bio_labeling.ipynb`: rule-based BIO labeler over `data/normalized/intent_dataset_normalized.jsonl`'s `text` field (disfluency markers are preserved there; `text_normalized` only strips punctuation, so neither field is a ready-made label) using a 5-tag taxonomy: `IP` (filler), `FS` (false start), `RC` (repeat), `RP`/`RM` (reparandum/repair pair). `label_core(toks, raw_toks)` does a single left-to-right pass tagging `FS` (hyphen-truncated token immediately followed by its completed form, e.g. `na- nasi`), `RC` (exact adjacent word duplicate, excluding `VOCATIVES` like `kak`/`bang`/`mas` so direct address never misfires as a repeat), and standalone `IP` fillers (`eh`, `anu`, `um`, `hmm`, `ee`, `em`). `apply_repair_bridges(toks, raw_toks, labels)` runs a second pass restricted to `", <bridge>,"` patterns where both flanking tokens are numerals with different values (e.g. `dua, eh, tiga`) — corpus inspection showed `anu`-bridges and most `eh`-bridges are pure mid-clause hesitation with no actual correction, so unrestricted bridge detection produced false `RP`/`RM` tags on cancel/query sentences (e.g. `mbak, anu, mie goreng...dibatalin` wrongly tagging `mbak` as `RP`); restricting to numeral swaps leaves only the 2 genuine repairs in the corpus. `validate_bio` auto-corrects orphan `I-` tags to `B-` (guards against future heuristic changes, though current logic never produces orphans). Labels align to `LazarusNLP/NusaBERT-base` (IndoBERT continued-pretrained on Indonesian + regional languages incl. Javanese — chosen over plain IndoBERT for Ordo's Javanese-accented speech) WordPiece tokens via `align_labels_with_tokens` (`word_ids()`-based, first-subword convention, `max_length=64`, `-100` for continuations/specials). `dominant_type(labels)` picks each record's rarest present tag (priority `FS > RP > RM > RC > IP > fluent`) for stratification; `train_test_split` runs twice (80/20, then the 20% into 10/10), so classes need >=4 occurrences to survive both splits — `MIN_STRATIFY_COUNT = 4` forces `FS`(3)/`RP`(2)/`RM`(2) entirely into train rather than crashing on a 1-member class. Writes `data/disfluency/{train,val,test}.jsonl` (1601/199/200) and `label_map.json` (7 labels: `O`, `B-IP`, `B-FS`, `B-RC`, `I-RC`, `B-RP`, `B-RM`).
- `notebooks/5.1_disfluency_repair_seq2seq.ipynb`: fine-tunes `Wikidepia/IndoT5-small` (`AutoModelForSeq2SeqLM`) on `data/disfluency_repair/{train,val,test}.jsonl` (`text -> text_normalized` pairs), treating disfluency repair as text-to-text generation rather than BIO span tagging, since some repairs are replacements (self-corrections like "dua, eh, tiga") not pure deletions. Uses `Seq2SeqTrainer`/`Seq2SeqTrainingArguments` with `DataCollatorForSeq2Seq`, `MAX_INPUT_LEN`/`MAX_TARGET_LEN=64`, runs on `mps`. Saves to `models/disfluency_repair_indot5_small`.

- `notebooks/4.2_ner_training.ipynb`: full IndoBERT entity-NER training pipeline mirroring `3.2_bio_labeling.ipynb`'s structure. `compute_label_counts`/inverse-frequency `class_weights` feed a `WeightedTrainer(Trainer)` (overridden `compute_loss`) fine-tuning `AutoModelForTokenClassification.from_pretrained("indobenchmark/indobert-base-p1", num_labels=19, ...)` over the 9-tag entity schema (18 labels + `O`). `TrainingArguments`: AdamW, `learning_rate=2e-5`, `warmup_ratio=0.1` linear schedule, `weight_decay=0.01`, 5 epochs, batch size 16, eval/save per epoch, `load_best_model_at_end` on val F1. `compute_metrics` scores entity-level precision/recall/F1/accuracy via `seqeval` after stripping `-100` positions. Saves to `models/indobert-ner-bio-final`. `trainer.predict(test_dataset)` reports per-entity-type breakdown on the held-out test split (rare tags `REMOVE`/`TABLE` flagged as most likely weak). `predict_entities(text)` reloads the saved model standalone, tokenizes raw text, decodes first-subword predictions back to word-level BIO tags, and extracts contiguous `B-X`/`I-X` spans as `{label, text, start, end}` — the structured order format downstream LangGraph nodes will consume.
- `notebooks/4.1_ner_labeling.ipynb`: entity-NER labeling pipeline using a local LLM (`mlx-community/Qwen2.5-7B-Instruct-4bit` via `mlx_lm.load`/`generate`) instead of rule-based span matching, since dish/drink names in the dataset go beyond `data/menu.json`'s 11 items. `label_sentence(text, max_retries=2)` prompts the model with a fixed 9-tag schema (`DISH`, `QUANTITY`, `MODIFIER`, `SIZE`, `DRINK`, `ADD_ON`, `REMOVE`, `TABLE`, `TAKE_AWAY`) and `parse_entities(raw)` validates the JSON span-list response, retrying on parse failure. Writes `data/ner_dataset_bio.jsonl`. Test results: F1=0.952, precision=0.939, recall=0.966, accuracy=0.987; `IM` f1=0.988, `RM` f1=0.870 (weaker — mostly `B-RM`/`I-RM` boundary confusion). Adds `models/` to `.gitignore`.

- `notebooks/3.2_bio_labeling.ipynb`: `AutoModelForTokenClassification.from_pretrained(MODEL_NAME, num_labels=5, id2label=..., label2id=...)` loads `indobenchmark/indobert-base-p1` with a freshly initialized classification head (123,854,597 total/trainable params, full fine-tune). - `notebooks/3.2_bio_labeling.ipynb`: `compute_label_counts(jsonl_path)` tallies train-split label ids (excluding `-100`), then `class_weights = total_tokens / (n_classes * count_c)` (inverse-frequency) feeds `CrossEntropyLoss(weight=class_weights, ignore_index=-100)` — token labels skew heavily toward `O` (~74%) with `I-IM` nearly absent (51/8521 train tokens), so unweighted loss would let the model collapse toward always predicting `O`.
- `notebooks/3.2_bio_labeling.ipynb`: `train_test_split` (scikit-learn) splits the 1100 aligned records 80/10/10 into train/val/test, stratified by `disfluency_type` so rare classes (`insertion`=37, `filled_pause`=57) keep proportional representation in every split rather than landing entirely in one partition. Outputs `data/disfluency_dataset_{train,val,test}.jsonl` (880/110/110 records). Adds `scikit-learn` (`^1.9.0`) to `pyproject.toml`/`poetry.lock`.
- `notebooks/3.2_bio_labeling.ipynb`: `align_labels_with_tokens(tokens, labels)` aligns word-level BIO labels (`O`/`B-RM`/`I-RM`/`B-IM`/`I-IM`) to IndoBERT (`indobenchmark/indobert-base-p1`) WordPiece subword tokens using `tokenizer(...).word_ids()` — first subword of each word keeps `label2id[label]`, continuation subwords and special tokens (`[CLS]`/`[SEP]`/padding) get `-100` so `CrossEntropyLoss` ignores them. Tokenizes with `is_split_into_words=True`, `truncation=True`, `max_length=64`, `padding="max_length"`. Runs over all 1100 records from `data/disfluency_dataset_bio.jsonl`, asserts `len(input_ids) == len(labels)` for every record (0 mismatches, 0 truncations), writes `input_ids`/`attention_mask`/`labels` per record to `data/disfluency_dataset_aligned.jsonl`.
- `notebooks/3.2_bio_labeling.ipynb`: builds `data/disfluency_dataset_bio.jsonl` from `data/synthetic_id_formal_informal_normalized.jsonl`. `detect_disfluency_type(tokens, text)` classifies each sentence into one of 8 surface patterns (`partial_word`, `repetition`, `substitution_repair`, `restart`, `insertion`, `filled_pause`, `discourse_marker`, `fluent`) via priority-ordered heuristics (hyphen-suffix regex, adjacent-span duplicate detection, repair-bridge/filled-pause position relative to sentence start), then dispatches to a matching `label_*(tokens, text)` function via the `LABELERS` dict to emit BIO tags: `B-RM`/`I-RM` for reparandum spans (abandoned/repeated content), `B-IM`/`I-IM` for interregnum spans (fillers, discourse markers, repair bridges). Because `detect_disfluency_type` only selects one type per sentence, `apply_filler_overlay(tokens, labels)` runs after the primary labeler to catch secondary `FILLED_PAUSES`/`DISCOURSE_MARKERS` tokens left as `O`, chaining adjacent matches into `B-IM`/`I-IM` spans instead of duplicate `B-IM` tags. `validate_bio(tokens, labels, record_id)` auto-corrects orphan `I-RM`/`I-IM` tags (no preceding `B-`/`I-` of the same type) to `B-` and unknown label values to `O`.
- `notebooks/3.1_build_disfluency_vocabulary.ipynb`: builds `data/vocabulary.json` via two-layer disfluency classification over `data/synthetic_id_formal_informal_normalized.jsonl`. Layer 1 (`infer_pos(token)`) is a morphological/phonological POS tagger (suffix rules for `-nya`/`-kan`/`-an`, verb prefixes `me-`/`ber-`/`ter-`/`di-`/`mem-`/`men-`, closed-class pronoun/address/particle/repair lookups, phonological interjection shape) returning `NOUN_VERB`/`VERB`/`NUM`/`PRON`/`ADDR`/`INTJ`/`PART`/`DM`/`PARTIAL`/`CONTENT`. Layer 2 (`compute_corpus_stats`, `is_statistical_filler`) computes per-token average PMI with neighboring tokens and Shannon entropy of sentence-position distribution (5 bins); tokens are flagged filler-like when `avg_pmi < PMI_THRESHOLD` (2.5) or `pos_entropy > ENTROPY_THRESHOLD` (1.2), filtered to `freq >= MIN_FREQ` (5). `build_vocabulary(...)` combines both layers: `INTJ`→`filled_pauses`, `DM`→`repair_bridges`, `PART`→`discourse_markers` (high confidence), and `CONTENT` tokens additionally flagged by Layer 2 are added to `discourse_markers` (low confidence). Includes `run_pos_tests()` regression suite covering novel interjections and content-word non-matches.
- `notebooks/generate_audio_dataset.ipynb`: TTS audio generation pipeline using `ChatterboxTTS` (`grandhigh/Chatterbox-TTS-Indonesian` checkpoint via `hf_hub_download`, running on `device="mps"`). For each record in `data/synthetic_id_formal_informal_normalized.jsonl`, calls `model.generate(text_normalized, audio_prompt_path=...)` with a style-specific voice prompt (`VOICE_PROMPTS = {"formal": "example1.wav", "informal": "example2.wav"}`), saves output to `data/audio/{id:04d}_{style}.wav` via `ta.save`, and writes a per-record manifest (`id`, `audio_path`, `text`, `text_normalized`, `style`, `domain`, `voice_prompt`) to `data/synthetic_id_formal_informal_manifest.jsonl`. Gated by `LIMIT` (`None` runs the full dataset). Notebook requires a separate `.venv-tts` kernel since `chatterbox-tts` pins `numpy>=2.0`/`torch==2.6.0`, conflicting with the main project's `onnxruntime==1.18.1` pin.
- `edge-tts` (`>=7.2.8,<8.0.0`) dependency added to `pyproject.toml`/`poetry.lock`.
- `transformers (>=5.12.1,<6.0.0)`, `torch (>=2.12.1,<3.0.0)`, `accelerate (>=1.14.0,<2.0.0)`, `seqeval (>=1.2.2,<2.0.0)`, `evaluate (>=0.4.6,<0.5.0)` dependencies added to `pyproject.toml`/`poetry.lock` for upcoming NER training/eval work.
- `data/synthetic_id_formal_informal.jsonl`: 1100 synthetic Indonesian restaurant-table utterances (`id`, `text`, `style`: formal/informal, `domain`: restoran), 550/550 formal-informal split, covering order placement, order changes/cancellations, and formal disfluency markers (e.g. "anu", "em", "jadi gini"); quantities mixed digit and word forms ("2 porsi" / "dua porsi"). Scoped to post-seating table interactions only (no reservation/payment/complaint utterances).
- Batch FLEURS (`google/fleurs`, `id_id` validation split) transcription cell: loads dataset via `load_dataset(..., split="validation")` with `Audio(decode=False)`, runs each sample through a separate `WhisperModel("large-v3-turbo", device=device, compute_type=compute_type)` instance, and writes `id`/`audio_path`/`reference`/`hypothesis` records to `data/fleurs_validation_transcripts.jsonl` for offline accuracy/normalization scoring against ground truth.
- Cell to persist `wakeword_debug_log` entries as newline-delimited JSON to `data/transcripts.jsonl` (append mode), capturing live wake-word transcription runs for later evaluation.
- `find_input_device_index(name_substring)`: resolves a PyAudio input device index by case-insensitive substring match against device name (e.g. `"MacBook Pro Microphone"`); raises `ValueError` if no match. Bound to `MACBOOK_MIC_INDEX` and passed as `input_device_index` everywhere a recorder is constructed.
- `make_wakeword_recorder(...)`: builds an `AudioToTextRecorder` (RealtimeSTT) configured with `wakeword_backend="oww"` and an explicit `openwakeword_model_paths=WAKE_WORD_MODEL_PATH` (`hey_jarvis_v0.1.onnx`) — explicit path avoids openWakeWord's default behavior of loading all bundled pretrained models (alexa, hey_mycroft, etc.), which would otherwise also trigger on non-target wake words. `wake_words_sensitivity=0.6`, `wake_word_activation_delay=0.0`, `wake_word_timeout=5.0`.
- `stream_transcribe_with_wakeword(stop_event, language=None, output=None, debug_log=None, max_partial_interval_s=5.0)`: replaces the prior `stream_transcribe` loop. Gates transcription behind wake-word detection via `on_wakeword_detected` callback (`on_wake()`), runs `recorder.text(on_final)` on a background daemon thread (`transcription_loop`), and re-arms wake-word listening after each utterance instead of exiting.
  - `on_realtime_update(text)`: emits partial transcript only when `text_changed` or `interval_elapsed >= max_partial_interval_s` (5s), so live feedback doesn't go silent on long pauses without new words.
  - `on_recorded_chunk(chunk)` / `is_capturing` flag: accumulates raw PCM chunks into `utterance_chunks` only while actively recording an utterance; uses a local `is_capturing` bool rather than reading `recorder.is_recording` because the chunk callback can fire on the recording thread before the `AudioToTextRecorder` constructor returns and binds `recorder`, which previously raced and raised `NameError`.
  - `on_final(text)`: on utterance end, concatenates `utterance_chunks` into an `int16` numpy array and runs the clipping check before printing the finalized utterance and debug-log entry (`debug_log.append(dict(idx=..., elapsed_s=..., text=...))`).
- `clipped_sample_ratio(audio_int16)`: returns fraction of int16 PCM samples at or beyond `INT16_CLIP_VALUE = 32767`. `stream_transcribe_with_wakeword` warns (does not reject) when ratio exceeds `CLIP_RATIO_WARN_THRESHOLD = 0.001` (0.1%), since Whisper transcribes clipped/hot-gain audio fluently but confidently wrong.
- Dataset loader and evaluation routine for speech-to-text accuracy testing.
- Real-time speech-to-text streaming using faster-whisper.
- `notebooks/2_text_normalization.ipynb`: text normalization pipeline for NER preprocessing — `remove_punctuation` (strips non-word/non-space chars via regex) and `normalize_numbers` (converts digit sequences to Indonesian words via `num2words(..., lang="id")`), composed in `normalize(text)`. - `notebooks/2_text_normalization.ipynb`: dropped the `remove_fillers` step (and its backing `FILLERS` list / `_FILLER_PATTERN` regex) from the `normalize(text)` pipeline — disfluency markers ("anu", "em", "jadi gini", etc.) are now left in `text_normalized` instead of being stripped.

### Fixed

- Pin `onnxruntime` to 1.18.1; newer versions silently broke openWakeWord's ONNX models (loaded inside `make_wakeword_recorder`), leaving wake word scores stuck near zero even on clear speech.
- `stream_transcribe_with_wakeword` re-arms wake-word listening (`emit(f"listening for wake word ({WAKE_WORD})...")`) after each utterance in `on_final`, instead of the recorder going quiet after one detection.

### Removed

- FLEURS batch transcription cells (`load_dataset`/`WhisperModel` loop and `fleurs_validation_transcripts.jsonl` output) from `notebooks/1_speech_to_text.ipynb` — batch transcription now lives in its own pipeline, separate from the live mic/wakeword notebook.
- Stale 10-sample TTS outputs (`data/audio/000{1..10}_{formal,informal}.wav`, `data/synthetic_id_formal_informal_manifest.jsonl`, `data/synthetic_id_formal_informal_normalized.jsonl`) — regenerated on next full run against the expanded 1100-row dataset and filler-preserving normalizer.
