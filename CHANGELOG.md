## [Unreleased]
### Added
- `notebooks/3.2_bio_labeling.ipynb`: builds `data/disfluency_dataset_bio.jsonl` from `data/synthetic_id_formal_informal_normalized.jsonl`. `detect_disfluency_type(tokens, text)` classifies each sentence into one of 8 surface patterns (`partial_word`, `repetition`, `substitution_repair`, `restart`, `insertion`, `filled_pause`, `discourse_marker`, `fluent`) via priority-ordered heuristics (hyphen-suffix regex, adjacent-span duplicate detection, repair-bridge/filled-pause position relative to sentence start), then dispatches to a matching `label_*(tokens, text)` function via the `LABELERS` dict to emit BIO tags: `B-RM`/`I-RM` for reparandum spans (abandoned/repeated content), `B-IM`/`I-IM` for interregnum spans (fillers, discourse markers, repair bridges). Because `detect_disfluency_type` only selects one type per sentence, `apply_filler_overlay(tokens, labels)` runs after the primary labeler to catch secondary `FILLED_PAUSES`/`DISCOURSE_MARKERS` tokens left as `O`, chaining adjacent matches into `B-IM`/`I-IM` spans instead of duplicate `B-IM` tags. `validate_bio(tokens, labels, record_id)` auto-corrects orphan `I-RM`/`I-IM` tags (no preceding `B-`/`I-` of the same type) to `B-` and unknown label values to `O`.
- `notebooks/3.1_build_disfluency_vocabulary.ipynb`: builds `data/vocabulary.json` via two-layer disfluency classification over `data/synthetic_id_formal_informal_normalized.jsonl`. Layer 1 (`infer_pos(token)`) is a morphological/phonological POS tagger (suffix rules for `-nya`/`-kan`/`-an`, verb prefixes `me-`/`ber-`/`ter-`/`di-`/`mem-`/`men-`, closed-class pronoun/address/particle/repair lookups, phonological interjection shape) returning `NOUN_VERB`/`VERB`/`NUM`/`PRON`/`ADDR`/`INTJ`/`PART`/`DM`/`PARTIAL`/`CONTENT`. Layer 2 (`compute_corpus_stats`, `is_statistical_filler`) computes per-token average PMI with neighboring tokens and Shannon entropy of sentence-position distribution (5 bins); tokens are flagged filler-like when `avg_pmi < PMI_THRESHOLD` (2.5) or `pos_entropy > ENTROPY_THRESHOLD` (1.2), filtered to `freq >= MIN_FREQ` (5). `build_vocabulary(...)` combines both layers: `INTJ`â†’`filled_pauses`, `DM`â†’`repair_bridges`, `PART`â†’`discourse_markers` (high confidence), and `CONTENT` tokens additionally flagged by Layer 2 are added to `discourse_markers` (low confidence). Includes `run_pos_tests()` regression suite covering novel interjections and content-word non-matches.
- `notebooks/generate_audio_dataset.ipynb`: TTS audio generation pipeline using `ChatterboxTTS` (`grandhigh/Chatterbox-TTS-Indonesian` checkpoint via `hf_hub_download`, running on `device="mps"`). For each record in `data/synthetic_id_formal_informal_normalized.jsonl`, calls `model.generate(text_normalized, audio_prompt_path=...)` with a style-specific voice prompt (`VOICE_PROMPTS = {"formal": "example1.wav", "informal": "example2.wav"}`), saves output to `data/audio/{id:04d}_{style}.wav` via `ta.save`, and writes a per-record manifest (`id`, `audio_path`, `text`, `text_normalized`, `style`, `domain`, `voice_prompt`) to `data/synthetic_id_formal_informal_manifest.jsonl`. Gated by `LIMIT` (`None` runs the full dataset). Notebook requires a separate `.venv-tts` kernel since `chatterbox-tts` pins `numpy>=2.0`/`torch==2.6.0`, conflicting with the main project's `onnxruntime==1.18.1` pin.
- `edge-tts` (`>=7.2.8,<8.0.0`) dependency added to `pyproject.toml`/`poetry.lock`.
- `transformers (>=5.12.1,<6.0.0)`, `torch (>=2.12.1,<3.0.0)`, `accelerate (>=1.14.0,<2.0.0)`, `seqeval (>=1.2.2,<2.0.0)`, `evaluate (>=0.4.6,<0.5.0)` dependencies added to `pyproject.toml`/`poetry.lock` for upcoming NER training/eval work.
- `data/synthetic_id_formal_informal.jsonl`: 1100 synthetic Indonesian restaurant-table utterances (`id`, `text`, `style`: formal/informal, `domain`: restoran), 550/550 formal-informal split, covering order placement, order changes/cancellations, and formal disfluency markers (e.g. "anu", "em", "jadi gini"); quantities mixed digit and word forms ("2 porsi" / "dua porsi"). Scoped to post-seating table interactions only (no reservation/payment/complaint utterances).
- Batch FLEURS (`google/fleurs`, `id_id` validation split) transcription cell: loads dataset via `load_dataset(..., split="validation")` with `Audio(decode=False)`, runs each sample through a separate `WhisperModel("large-v3-turbo", device=device, compute_type=compute_type)` instance, and writes `id`/`audio_path`/`reference`/`hypothesis` records to `data/fleurs_validation_transcripts.jsonl` for offline accuracy/normalization scoring against ground truth.
- Cell to persist `wakeword_debug_log` entries as newline-delimited JSON to `data/transcripts.jsonl` (append mode), capturing live wake-word transcription runs for later evaluation.
- `find_input_device_index(name_substring)`: resolves a PyAudio input device index by case-insensitive substring match against device name (e.g. `"MacBook Pro Microphone"`); raises `ValueError` if no match. Bound to `MACBOOK_MIC_INDEX` and passed as `input_device_index` everywhere a recorder is constructed.
- `make_wakeword_recorder(...)`: builds an `AudioToTextRecorder` (RealtimeSTT) configured with `wakeword_backend="oww"` and an explicit `openwakeword_model_paths=WAKE_WORD_MODEL_PATH` (`hey_jarvis_v0.1.onnx`) â€” explicit path avoids openWakeWord's default behavior of loading all bundled pretrained models (alexa, hey_mycroft, etc.), which would otherwise also trigger on non-target wake words. `wake_words_sensitivity=0.6`, `wake_word_activation_delay=0.0`, `wake_word_timeout=5.0`.
- `stream_transcribe_with_wakeword(stop_event, language=None, output=None, debug_log=None, max_partial_interval_s=5.0)`: replaces the prior `stream_transcribe` loop. Gates transcription behind wake-word detection via `on_wakeword_detected` callback (`on_wake()`), runs `recorder.text(on_final)` on a background daemon thread (`transcription_loop`), and re-arms wake-word listening after each utterance instead of exiting.
  - `on_realtime_update(text)`: emits partial transcript only when `text_changed` or `interval_elapsed >= max_partial_interval_s` (5s), so live feedback doesn't go silent on long pauses without new words.
  - `on_recorded_chunk(chunk)` / `is_capturing` flag: accumulates raw PCM chunks into `utterance_chunks` only while actively recording an utterance; uses a local `is_capturing` bool rather than reading `recorder.is_recording` because the chunk callback can fire on the recording thread before the `AudioToTextRecorder` constructor returns and binds `recorder`, which previously raced and raised `NameError`.
  - `on_final(text)`: on utterance end, concatenates `utterance_chunks` into an `int16` numpy array and runs the clipping check before printing the finalized utterance and debug-log entry (`debug_log.append(dict(idx=..., elapsed_s=..., text=...))`).
- `clipped_sample_ratio(audio_int16)`: returns fraction of int16 PCM samples at or beyond `INT16_CLIP_VALUE = 32767`. `stream_transcribe_with_wakeword` warns (does not reject) when ratio exceeds `CLIP_RATIO_WARN_THRESHOLD = 0.001` (0.1%), since Whisper transcribes clipped/hot-gain audio fluently but confidently wrong.
- Dataset loader and evaluation routine for speech-to-text accuracy testing.
- Real-time speech-to-text streaming using faster-whisper.
- `notebooks/2_text_normalization.ipynb`: text normalization pipeline for NER preprocessing â€” `remove_punctuation` (strips non-word/non-space chars via regex) and `normalize_numbers` (converts digit sequences to Indonesian words via `num2words(..., lang="id")`), composed in `normalize(text)`. Runs over `data/synthetic_id_formal_informal.jsonl` (1100 rows), writes `data/synthetic_id_formal_informal_normalized.jsonl` with a `text_normalized` field added per record.

### Changed
- `notebooks/2_text_normalization.ipynb`: dropped the `remove_fillers` step (and its backing `FILLERS` list / `_FILLER_PATTERN` regex) from the `normalize(text)` pipeline â€” disfluency markers ("anu", "em", "jadi gini", etc.) are now left in `text_normalized` instead of being stripped.

### Fixed
- Pin `onnxruntime` to 1.18.1; newer versions silently broke openWakeWord's ONNX models (loaded inside `make_wakeword_recorder`), leaving wake word scores stuck near zero even on clear speech.
- `stream_transcribe_with_wakeword` re-arms wake-word listening (`emit(f"listening for wake word ({WAKE_WORD})...")`) after each utterance in `on_final`, instead of the recorder going quiet after one detection.

### Removed
- FLEURS batch transcription cells (`load_dataset`/`WhisperModel` loop and `fleurs_validation_transcripts.jsonl` output) from `notebooks/1_speech_to_text.ipynb` â€” batch transcription now lives in its own pipeline, separate from the live mic/wakeword notebook.
- Stale 10-sample TTS outputs (`data/audio/000{1..10}_{formal,informal}.wav`, `data/synthetic_id_formal_informal_manifest.jsonl`, `data/synthetic_id_formal_informal_normalized.jsonl`) â€” regenerated on next full run against the expanded 1100-row dataset and filler-preserving normalizer.
